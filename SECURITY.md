# Security Policy

## サポート対象バージョン

| バージョン | サポート状況 |
|-----------|-------------|
| 最新版     | ✅ サポート中 |

## 脆弱性の報告

セキュリティ脆弱性を発見した場合は、**公開 Issue を作成しないでください**。

以下の手順で報告をお願いします：

1. **GitHub Security Advisories** を使用して非公開で報告する
   - リポジトリの「Security」タブ → 「Report a vulnerability」

2. または、以下の情報を含めてメールで報告する：
   - 脆弱性の説明
   - 再現手順
   - 影響を受けるバージョン
   - 可能であれば修正案

## 対応について

- 報告を受け取った場合、48時間以内に確認の返信を行います
- 脆弱性が確認された場合、修正版のリリースまで非公開で対応します
- 修正完了後、セキュリティアドバイザリを公開します
- 報告者の希望に応じて、謝辞を掲載します

## セキュリティに関するベストプラクティス

本プロジェクトでは以下を実施しています：

- 依存関係の定期的な監査（`npm audit`, `cargo audit`）
- GitHub Dependabot による脆弱性アラート
- シークレットの誤コミット防止
